🎯 Content-Nuke-Claude Command

Custom Command: blog-startai.md

# Blog Command for StartAI Tools
# Usage: /blog-startai

You are a technical content specialist writing for StartAI Tools, an AI development insights platform.

## Blog Configuration
BLOG_DIR="/home/jeremy/projects/blog/myblog/startaitools"
CONTENT_DIR="$BLOG_DIR/content/posts"
BUILD_COMMAND="hugo --gc --minify --cleanDestinationDir"
SERVE_COMMAND="hugo server -D --bind 0.0.0.0 --port 1313"
DEPLOY_BRANCH="master"
REMOTE_ORIGIN="origin"

## Content Analysis
Analyze the current development session:

1. **Technical Context**: Review git commits, code changes, and project files
2. **Session Summary**: Identify key technical achievements and insights
3. **Architecture Insights**: Extract system design patterns and decisions
4. **Implementation Details**: Highlight code examples and technical solutions
5. **Business Impact**: Connect technical work to business value

## Content Generation

Create a comprehensive technical blog post with:

### Front Matter (YAML)
```yaml
---
title: "Descriptive Technical Title (60-80 characters)"
date: {{ current_date }}
draft: false
tags: ["ai", "development", "architecture", "automation"]
categories: ["technical"]
description: "Compelling description (150-160 characters for SEO)"
author: "Jeremy Longshore"
featured_image: "/images/posts/{{ slug }}-featured.png"
series: "AI Development Series"
weight: 1
---
```

### Content Structure
1. **Executive Summary** (150-200 words)
   - Key technical achievement
   - Business impact
   - Main takeaways

2. **Technical Deep Dive** (800-1200 words)
   - Problem statement and context
   - Solution architecture
   - Implementation details with code examples
   - Testing and validation approach

3. **Architecture Insights** (300-500 words)
   - Design patterns used
   - Scalability considerations
   - Performance optimizations
   - Security implications

4. **Code Examples** (300-400 words)
   - Key code snippets with explanations
   - Configuration examples
   - Best practices demonstrated

5. **Lessons Learned** (200-300 words)
   - What worked well
   - Challenges encountered
   - Future improvements

6. **Conclusion** (100-150 words)
   - Summary of achievements
   - Next steps
   - Call to action

### SEO Optimization
- Target 2000-2500 words total
- Include relevant keywords naturally
- Use proper heading hierarchy (H2, H3)
- Add internal links to related posts
- Include code blocks with syntax highlighting

## File Operations

1. **Generate filename**: Use format `YYYY-MM-DD-descriptive-slug.md`
2. **Save to**: `$CONTENT_DIR/[filename]`
3. **Build site**: Run `$BUILD_COMMAND` in `$BLOG_DIR`
4. **Git workflow**:
   ```bash
   cd $BLOG_DIR
   git add .
   git commit -m "feat: add blog post [title]"
   git push $REMOTE_ORIGIN $DEPLOY_BRANCH
   ```

## Analytics Integration

After successful publication:
1. **Track post metrics**: Word count, read time, tags
2. **Update analytics**: Add to content database
3. **Generate X thread**: Create 3-5 tweet summary thread
4. **LinkedIn post**: Professional announcement (1200-1500 chars)

Before publishing, verify:

- [ ] SEO description is compelling
- [ ] Content flows logically
- [ ] Technical accuracy is verified

🚀 Deployment Pipeline

Automated Deployment with Netlify

StartAITools.com uses Netlify for continuous deployment:

# netlify.toml
[build]
  command = "hugo --gc --minify --cleanDestinationDir"
  publish = "public"

[build.environment]
  HUGO_VERSION = "0.150.0"
  NODE_VERSION = "18"

[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-XSS-Protection = "1; mode=block"
    X-Content-Type-Options = "nosniff"
    Referrer-Policy = "strict-origin-when-cross-origin"

# Redirect rules
[[redirects]]
  from = "/feed.xml"
  to = "/index.xml"
  status = 301

GitHub Actions Alternative

# .github/workflows/hugo.yml
name: Deploy Hugo to GitHub Pages

on:
  push:
    branches: [ master ]
  pull_request:
    branches: [ master ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
      with:
        submodules: true
        fetch-depth: 0

    - name: Setup Hugo
      uses: peaceiris/actions-hugo@v2
      with:
        hugo-version: '0.150.0'
        extended: true

    - name: Build
      run: hugo --gc --minify --cleanDestinationDir

    - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      if: github.ref == 'refs/heads/master'
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./public

📊 Analytics Integration

Automatic Post Tracking

Content-Nuke-Claude automatically tracks all published posts:

# analytics_helpers.py integration
def auto_add_blog_post(file_path, site='startaitools'):
    """Track blog post in analytics database"""

    # Extract front matter
    with open(file_path, 'r') as f:
        content = f.read()

    # Parse YAML front matter
    front_matter = extract_front_matter(content)

    # Calculate metrics
    word_count = count_words(content)
    read_time = calculate_read_time(word_count)

    # Insert into database
    insert_blog_post({
        'site': site,
        'title': front_matter.get('title'),
        'slug': generate_slug(front_matter.get('title')),
        'published_date': front_matter.get('date'),
        'word_count': word_count,
        'read_time': read_time,
        'tags': ','.join(front_matter.get('tags', [])),
        'categories': ','.join(front_matter.get('categories', [])),
        'file_path': file_path
    })
